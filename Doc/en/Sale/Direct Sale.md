# Direct Sale Module

The Direct Sale feature is used to manage immediate product sales. It allows users to create, manage, and approve sales transactions efficiently.

## Accessing Direct Sale Dashboard

>     Navigate to Sale > Direct Sale.

<img src="../images/direct sale dashboard.png" alt="direct sale dashboard" style="border-radius: 10px; width: 70%; height: 70%; border: 0.5px solid #333;">

The Direct Sale Dashboard will appear, displaying: All direct sales categorized as Draft or Approved. There will be a Search Bar for filtering specific records.

## Report Options in Direct Sale

In the dashboard, you can export sales data with customized options:

<div style="text-align:left;">

        <li><strong>Select Shape:</strong> Landscape or Portrait.</li>

## Creating a New Direct Sale

Click Create New in the top-right bar to open the Direct Sale form. The process includes:

<img src="../images/ceate new direct sale.png" alt="ceate new direct sale" style="border-radius: 10px; width: 70%; height: 70%; border: 0.5px solid #333;">

### Document Information

Document Number Series: Auto-generated by the system. Configure via the ðŸ‘‰ [Document Series Settings](<direct purchase.md>).<br>
Date: Specify the sale date.

### Price and Store Details

Price Type: Choose from Wholesale, Retail, or Regular. Configure via ðŸ‘‰ [Price Type Settings](<direct purchase.md>).<br>
Store: Select the store for the sale. Configure via ðŸ‘‰ [Store Settings](<direct purchase.md>).<br>

### Customer Information
Customer: Select the customer. Add customers via ðŸ‘‰ [Customer Menu](<direct purchase.md>).<br>
Displays any Due Amount for the selected customer.<br>
If the customer has associated tax (e.g., UAE VAT, India GST), the tax column will appear in the item section.

### Salesperson and Project
Salesperson: Assign a salesperson. Create salespersons via ðŸ‘‰ [Salesperson Settings](<direct purchase.md>).<br>
Project: Link the sale to a specific project. Configure via ðŸ‘‰ [Project Settings](<direct purchase.md>).<br>

<img src="../images/fields in direct sale.png" alt="fields in direct sale" style="border-radius: 10px; width: 70%; height: 70%; border: 0.5px solid #333;">

### Item Selection
Item: Select the item from the dropdown list. The list displays Current Stock, Reorder Quantity, and Average Rate for the item. If the item doesnâ€™t exist, create it via ðŸ‘‰ [Item Menu](<direct purchase.md>).<br>
Enter Quantity, and the unit and rate will auto-fill based on the item's configuration. You can manually adjust the rate if required. Add more items by clicking Add Line.

<img src="../images/item in direct sale.png" alt="item in direct sale" style="border-radius: 10px; width: 70%; height: 70%; border: 0.5px solid #333;">

### Financial Details
Subtotal: Automatically calculated.<br>
Discount: Apply a discount if applicable.<br>
Adjust Amount: Add or subtract adjustments.<br>
Add additional details such as Reference Number, Statement Reference, PO Number, PO Date, SL Number for Machines, and Customer Branch.

### Payment Options
Select a Payment Type (Cash, Bank, Cheque, or Credit):

#### Credit
Payment Term & Due Date: Options include:<br>
Due End of the Month.<br>
Due End of Next Month.<br>
Due on Receipt.<br>
Custom.<br>
The due date updates automatically based on the selected term.
Once payment details are entered, click Add.

#### Cash
Cost Center: Configure via ðŸ‘‰ [Cost Center Settings](<direct purchase.md>).<br>
Cash Account: Configure via ðŸ‘‰ [Cash Account Settings](<direct purchase.md>).<br>
Cash Repository: Configure via ðŸ‘‰ [Cash Repository Settings](<direct purchase.md>).<br>
Once payment details are entered, click Add.

#### Cheque
Cost Center: Configure via ðŸ‘‰ [Cost Center Settings](<direct purchase.md>).<br>
Bank: Configure via ðŸ‘‰ [Bank Settings](<direct purchase.md>).<br>
Enter Cheque Number and select Cheque Date.
Once payment details are entered, click Add.

#### Bank
Cost Center: Configure via ðŸ‘‰ [Cost Center Settings](<direct purchase.md>).<br>
Bank: Configure via ðŸ‘‰ [Bank Settings](<direct purchase.md>).<br>
Add Bank Posted Date, Instrument Code, and Bank Transaction Code.<br>
Once payment details are entered, click Add.

### Saving and Exporting Direct Sale

After entering all details, click Save.<br>
A PDF of the direct sale will be generated.<br>

### Post-Save Options

The top-right bar provides the following options after saving:

<div style="text-align:left;">
    <ul>
        <li><strong>Back:</strong> Return to the dashboard.</li>
        <li><strong>Preview Settings:</strong> Customize the document preview.</li>
        <li><strong>Previous Preview:</strong> View previous document versions.</li>
        <li><strong>New Create:</strong> Start a new direct purchase.</li>
        <li><strong>Edit:</strong> Modify the document (before approval only).</li>
        <li><strong>Delete:</strong> Remove the document.</li>
        <li><strong>Print:</strong> Print the purchase record.</li>
        <li><strong>Download:</strong> Export the document.</li>
        <li><strong>Download Without Header and Footer:</strong> Export a clean version.</li>
        <li><strong>Go To > Stock Ledger:</strong> View the stock movement ledger.</li>
        <li><strong>Go To > Provisional Account Ledger:</strong> View account details.</li>
    </ul>
</div>

### Approval Workflow

>     Approve the sale by clicking Action > Approve.

Once approved:<br>
The status changes from Draft to Approved.<br>
The Approve button becomes Withdraw.<br>
Editing the record is disabled.
